# ペルソナとユーザー要求（仮説）
作成日: 2026-02-05

## ペルソナ
- 年齢層: 20〜40代
- 旅行頻度: 年に1〜3回
- 旅行人数: 1人〜3人
- 旅行スタイル: 主要スポットは決めるが当日も柔軟に動きたい
- 利用端末: 事前はPC、当日はスマホ

## 現状の行動（仮説）
- カレンダー、メモ、地図ブックマークが分散している
- 共有は最小限でよく、必要な人だけに見せたい
- 旅行当日は「今どこへ行くか」の確認に時間がかかる
- 旅行後の精算はメッセージで手動集計している

## 求めていること（ユーザー要求）
- 情報を1か所にまとめ、すぐ参照できる
- タイムラインの入れ替えや時間調整が簡単
- 未確定の候補を一時置きできる
- 地図リンクとメモをすぐ開ける
- オフラインでも最低限の閲覧が可能
- 共有は「閲覧のみ」など最小権限で済ませたい
- 旅行後に写真とメモで振り返れる
- 友人と計画をリアルタイムで更新できる
- 立替金を正確に記録し、割り勘を早く終えたい

## 課題（仮説）
- 旅行情報が散在しており、確認の導線が複雑
- 旅程の変更が面倒で、更新しなくなる
- 共有したいが、他人に編集されるのは避けたい
- 旅行アプリが多機能で、必要な情報に辿り着きにくい
- 写真や思い出がバラバラに保存され、振り返りにくい
- 精算ルールが曖昧で、支払い漏れや不公平感が出る
