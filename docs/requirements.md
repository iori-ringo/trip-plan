# 旅行プランナー 要件定義（v0.2）
作成日: 2026-02-05

## 目的
- 旅行の計画情報を1か所に集約し、当日の迷いを減らす
- 旅程の組み立てと見直しを短時間でできるようにする
- オフラインでも閲覧できるようにする
- 旅行後のふりかえりや共有をしやすくする

## ゴール
- 旅行単位で計画を作成・編集・保存できる
- 日付ごとのタイムラインで行動を整理できる
- スポット情報やメモ、地図リンクを簡単に参照できる
- JSONでエクスポートしてバックアップできる
- 旅行の写真やハイライトをまとめられる
- 友人と計画をリアルタイムで更新できる
- 立替金を記録し、旅行内で割り勘精算できる

## スコープ（MVP）
- 旅行管理（作成・編集・削除・一覧）
- 日程タイムライン編集（並び替え・時間設定）
- スポット/アクティビティ詳細の管理
- 検索/フィルタ（旅行内）
- ローカル保存＋インポート/エクスポート
- ふりかえり（旅行終了後のみ表示）

## 機能優先度（要約）
- MVP: 旅程編集、未割当整理、検索、ローカル保存、JSONエクスポート
- Next: 共有用の閲覧モード、費用集計、チェックリスト、写真管理（クラウド）、割り勘
- Stretch: リアルタイム共同編集（ログイン必須 + WebSocket）、交通手段の自動提案、AIによる行程サジェスト

## 非スコープ（MVPではやらない）
- 予約/決済/チケット連携
- 交通経路の最適化や自動提案
- 外部API連携（地図API、レビューAPIなど）
- ログインなしでの共同編集

## 想定ユーザー
- 1人〜少人数の旅行を計画する人
- 事前にある程度計画を立て、当日は確認したい人
- メモや地図リンクを分散させたくない人
- 旅行費用の精算をなるべく簡単に済ませたい人

## ペルソナ要約
- 旅行は年に数回、計画は立てるが当日も柔軟に動きたい
- 予定はカレンダーやメモに分散しがちで、当日の確認が面倒
- 共有は最小限でよく、必要な人にだけ見せたい
- 旅行後に写真や学びをまとめて振り返りたい

## ユーザー課題（要約）
- 情報が散在していて確認に時間がかかる
- タイムラインの並び替えや時間調整が面倒
- 予約情報や地図リンクが埋もれて見つからない
- 旅行後のふりかえりが断片的で記録が残らない
- 誰がいくら立替えたかが曖昧で、精算に時間がかかる

## 主要ユーザーフロー
1. 旅行を作成
2. 期間を設定し、日程が自動生成される
3. 各日にスポットを追加して時間や順序を調整
4. 必要に応じて検索やフィルタで再確認
5. エクスポートしてバックアップ

## 非機能要件
- パフォーマンス
- 1旅行あたり1,000件のItemを扱っても操作が破綻しない
- 日程の並び替えが体感で滑らかに行える

- アクセシビリティ
- キーボード操作で主要機能が完結できる
- ラベルとコントラストを適切に設定する

- レスポンシブ
- スマホ幅で主要フローが完結できる
- PCでは2カラム以上の編集体験を提供

- 信頼性
- リロード後もデータが保持される
- インポート失敗時はデータ破損しない

- セキュリティ
- 共同編集はログイン済みユーザーのみアクセス可能
- 共有リンクは失効できる

## UX原則
- 1操作で1目的が達成できるUI
- 迷ったら「未割当」へ逃がせる
- データ消失は明示的な確認が必要

## MVP受け入れ条件
- 旅行の作成から日程編集、保存、再読み込みが通る
- 旅行内の検索/フィルタで対象Itemが特定できる
- JSONエクスポート/インポートが正しく動作する
- 旅行終了後のみふりかえり画面が表示される

## 将来拡張候補
- 共有リンク（読み取り専用）
- 地図の埋め込みビュー
- 交通手段や移動時間の管理
- 予算管理/割り勘の高度化
- AIによる行程の提案

## 決定済み事項（2026-02-06）
- リアルタイム共同編集はログイン必須
- 同期方式はWebSocket
- 写真保存はクラウド
- ふりかえりは旅行終了後のみ表示
- 割り勘機能を導入する
