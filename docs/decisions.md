# 決定ログ
作成日: 2026-02-06

## 2026-02-06
- リアルタイム共同編集はログイン必須
- 同期方式はWebSocket
- 写真保存はクラウド
- ふりかえりは旅行終了後のみ表示
- 割り勘機能を導入

## 影響
- 共同編集には認証・認可の基盤が必要
- WebSocket運用のため接続管理と再接続処理が必要
- 写真はクラウド保存のためアップロード失敗時リトライ設計が必要
- ふりかえり画面には終了判定ロジックが必要
- 割り勘には参加者管理と精算ロジックが必要

## 未決事項
- 認証プロバイダ（例: Clerk/Auth0/自前）
- クラウドストレージ（例: S3/Cloudflare R2/Firebase Storage）
- WebSocket基盤（例: Socket.IO/Pusher/Ably）
- 割り勘の端数処理ルール（切り上げ/切り捨て/最近傍）
