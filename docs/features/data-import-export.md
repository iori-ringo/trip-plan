# データ保存/インポート/エクスポート 要件

## 概要
旅行データをローカルで保持し、バックアップできる。

## 目的
- リロードや再訪時もデータを維持する
- 端末移行や復元が可能になる

## 仕様
- 変更は自動でローカル保存される
- エクスポートはJSONファイルを生成する
- インポートはJSONを読み込み旅行として追加する
- 破損ファイルは読み込みエラーとして扱う

## 受け入れ基準
- ブラウザ再読み込み後もデータが残る
- エクスポートでJSONがダウンロードできる
- インポートで旅行が増える

## エッジケース
- スキーマ不一致はエラーメッセージを表示
- 同名旅行が存在しても上書きしない
